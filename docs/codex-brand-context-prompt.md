# Codex Brand Context Prompt

Use this prompt when asking Codex or another coding assistant to work in this repository.

```text
You are working in the nsn-mobile repository.

Brand context:
- North Shore Nights (NSN) is the current Sydney/North Shore alpha pilot.
- SoftHello is the future broader/global product direction, not a live or production-ready product.
- SoftShore is historical naming exploration unless a future scoped naming review reopens it.
- Tagline: Small meetups for big moments.
- The repo may remain named nsn-mobile.
- Do not blindly rename repositories, package names, folders, bundle identifiers, route names, code symbols, or every NSN reference.
- Repo/package/app identifier renames must be separate future tasks with explicit scope and validation.

Copy direction:
- Public-facing pilot copy should use NSN or North Shore Nights.
- Broader product and shared principle copy may use SoftHello only as future-facing direction.
- Do not imply SoftHello is currently launched, available, production-ready, or supported by live systems.
- Keep language warm, calm, respectful, and practical.
- Prefer small gatherings, reaching out, reconnecting, privacy, consent, comfort, and belonging.
- Avoid making the product sound like nightlife, dating, generic networking, performance, popularity, or loud social media.
- Do not overpromise production safety, verification, moderation, matching, or emergency support before those systems are real.

When editing:
- Follow existing Expo/React Native/TypeScript patterns.
- Keep changes scoped.
- Update docs when brand-facing assumptions change.
- Run the repository's validation commands before claiming completion.
```
